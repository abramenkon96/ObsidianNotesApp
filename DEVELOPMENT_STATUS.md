# Статус разработки: Obsidian Notes Android

## 1. Текущее состояние
Код полностью готов к работе, архитектура выстроена по паттерну MVVM. Основная проблема — **отсутствие локального Android SDK 34** и ошибки в облачной сборке (вероятно, из-за конфликтов в ресурсах или кэша Gradle).

## 2. Что реализовано (Done)
- **UI (Compose):** Список заметок с раскрывающимися карточками, поиск, выбор папки через SAF.
- **Интеграция Markdown:** Использование библиотеки Markwon для рендеринга `.md` контента прямо в списке.
- **БД (Room):** Настроен FTS4 для быстрого полнотекстового поиска.
- **ViewModel:** Вся логика отделена от Activity, используется `StateFlow` для реактивности.
- **Синхронизация:** `SyncManager` умеет рекурсивно сканировать папки через `DocumentFile`.

## 3. Технические детали изменений
- **MainActivity.kt**: Переписана на Jetpack Compose. Удалена прямая работа с БД.
- **MainViewModel.kt**: Центральный узел логики. Обрабатывает поиск и синхронизацию.
- **SyncManager.kt**: Исправлены ошибки типизации и логика обхода файлов.
- **Resources**: Добавлены `themes.xml` (values/values-night) и `ui/theme/*.kt`.

## 4. Почему не собрался APK (Known Issues)
- **Локально:** На хосте нет SDK 34. Попытка использовать SDK от Unity (API 29) провалилась из-за требований Material 3 к API 34.
- **GitHub Actions:** Сборка падает. Вероятные причины:
  1. Несоответствие версий `gradle-wrapper` и `AGP` (Android Gradle Plugin).
  2. Ошибки в `libs.versions.toml` (версия KSP может конфликтовать с Kotlin 2.0.0).

## 5. Инструкции для следующего шага
1. **Проверить `gradle/libs.versions.toml`**: убедиться, что версии Kotlin, Compose и KSP строго совместимы.
2. **Локальная сборка**: рекомендуется установить Android Studio, она сама докачает нужный SDK 34.
3. **GitHub CI**: Проверить логи шага "Build with Gradle". Если ошибка "License not accepted", нужно обновить хэши лицензий в воркфлоу.
