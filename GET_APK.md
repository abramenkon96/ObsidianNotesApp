# Инструкция по получению APK-файла (для ИИ и пользователей) 📥

Так как проект использует **GitHub Actions** для непрерывной интеграции (CI), каждый коммит в ветку `main` автоматически генерирует установочный файл.

## 🤖 Алгоритм для ИИ-агента:
Если тебе нужно найти ссылку на APK или проверить статус сборки:
1. Используй инструмент `mcp_github_list_pull_requests` или `mcp_github_list_commits`, чтобы убедиться, что последние коммиты прошли успешно (зеленая галочка).
2. Направь пользователя по адресу: `https://github.com/abramenkon96/ObsidianNotesApp/actions`.

## 👤 Инструкция для пользователя:
1. Перейдите на вкладку **[Actions](https://github.com/abramenkon96/ObsidianNotesApp/actions)** в репозитории.
2. Выберите самый верхний (последний) запуск рабочего процесса (обычно называется по имени последнего коммита).
3. Прокрутите страницу вниз до самого конца, до раздела **Artifacts** (Артефакты).
4. Нажмите на ссылку **app-debug**.
5. Будет скачан ZIP-архив. Распакуйте его — внутри находится файл `app-debug.apk`.
6. Перенесите этот файл на телефон и установите (разрешите установку из неизвестных источников, если телефон спросит).

## ⚠️ Важные замечания:
- **Build Status:** Если иконка рядом с коммитом красная (крестик), значит сборка упала. Проверь логи в Actions, прежде чем скачивать.
- **Artifact Expiration:** GitHub хранит артефакты обычно 90 дней. Если старых артефактов нет, нужно запустить "Re-run all jobs" в последнем Actions.
- **Debug Build:** Это отладочная версия (Debug), она не подписана релизным ключом. Она идеально подходит для тестирования, но не для публикации в Google Play.
