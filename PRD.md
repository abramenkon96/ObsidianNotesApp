# Project Requirements Document (PRD): Android Markdown Notes

## 1. Идея продукта
Создание Android-приложения для ведения заметок, которое работает напрямую с файловой системой (формат `.md`), использует YAML Frontmatter для метаданных (совместимость с Obsidian) и имеет глубокую интеграцию с системными уведомлениями.

## 2. Ключевые функции
- **Markdown-ориентированность:** Чтение и редактирование `.md` файлов.
- **Двухрежимный UI:** Переключение Чтение/Редактирование по двойному клику или иконке.
- **Интерактивные списки:** Возможность вычеркивать пункты (`- [ ]` -> `- [x]`) прямо в режиме чтения.
- **Организация:** Группировка заметок и цветовая маркировка.
- **Архивация:** Вместо удаления — перенос в папку `.archive`.
- **Pinned Notifications:** Закрепление заметок в шторке уведомлений как "неудаляемое" (ongoing). Каждая заметка — отдельное уведомление.
- **Гибкость хранения:** Выбор корневой папки через Storage Access Framework (SAF), настройка путей для заметок и архива.

## 3. Технический стек
- **Kotlin + Jetpack Compose**
- **Architecture:** MVVM + Clean Architecture
- **File Access:** Android Storage Access Framework (SAF)
- **Data Persistence:** YAML Frontmatter (SnakeYAML), Metadata cache (Room - опционально для поиска)
- **Markdown Engine:** Markwon / Compose-Markdown

## 4. Структура данных (File-first)
Каждый файл должен содержать заголовок в формате YAML:
```yaml
---
title: "Название заметки"
group: "Проекты"
color: "#AABBCC"
pinned: true
status: "active"
---
Текст заметки в формате Markdown...
- [ ] Пункт списка
```

## 5. План разработки

### Этап 1: Инициализация и Файловая система
- Создание Android проекта.
- Реализация SAF (выбор папки, права доступа).
- Разработка менеджера файлов (сканирование, чтение метаданных).

### Этап 2: Интерфейс и Редактор
- Главный экран: Список (LazyColumn) с фильтрами.
- Экран заметки: Markdown рендерер + TextField редактор.
- Логика двойного клика.

### Этап 3: Уведомления и Сервисы
- Foreground Service для удержания уведомлений.
- Notification Manager: создание и обновление кастомных уведомлений для заметок.

### Этап 4: Настройки и Архив
- Экран настроек: выбор путей `/notes` и `/.archive`.
- Логика перемещения файлов при архивации.

---
## 6. Примечания для ИИ-агентов
- Весь "интеллект" приложения должен опираться на содержимое файлов. 
- Избегать жесткой привязки к внутренней базе данных. 
- При записи файла ВСЕГДА сохранять структуру YAML.
